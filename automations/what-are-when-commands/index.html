Introduction to automation

Welcome to the automation documentation for ComfyAI.

What is this?​

Here you'll learn about ComfyAI's real advantage over other bots: the ability to automate anything.

Comfy can automate any action on Discord and the internet to help you manage your server and transmit information between external services. The way this works is Comfy "listens" for an event to happen, then runs an action in response.

You choose the "events" and the "actions" by defining short snippets of instructions we call "When Commands". That's a mouthful, but think of it as "I'm listening for when something happens, then I want to do something in response".

Introductory examples​

Let's create a comparison to ensure you understand the principal:

  • "When my dog scratches at the door, I let him into the house"

The event is your dog scratching at the door, and the action is letting him in. Now imagine these examples on Discord:

  • "When a user reacts to a specific message, give them a role"

  • "When a user joins my server, send a welcome message in the welcome channel"

  • "When a user sends a message with profanity, block the message, and warn that user to stop"

  • "When a user joins my Valheim server, give them a role in Discord"

  • "When the price of Bitcoin reaches $100k, post a message in a specific channel in Discord"

  • "When a user sends a message in a specific channel, post it inside my Minecraft server's in-game chat window"

This should be making more sense now. The first part of the sentence starting with "when" is the "event" that Comfy "listens" for. When it sees it, it performs an action in response.

In the following section we're going to show you how simple it is to use Comfy to create these "when listeners".
