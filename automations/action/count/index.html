Count

Action expression​

The count action increments a counter key.

A counter has two parts, the count action and the counter listener. When a count action increments the count of a specific key to the set amount, the counter listener triggers an action.

The Count required to trigger the Counter Listener's action is always set with this Count Action. This means different counts can be used to trigger the Counter Listener.

Possible arguments​

Key​

Defines the key name of the count action to trigger the counter listener.

This argument takes a keyword value.

ValueDescription
KeywordThe same name as the counter listener to connect the action. Must not contain a space.

Trigger​

Defines the count that must be achieved to trigger the counter listener.

This argument takes a number value.

ValueDescription
NumberThe count you want to trigger the Counter Listener's action at

Example​

when reaction emote= <:UwU:840311365813010492> -> count key= my_uwu_counter trigger= 10

When the reaction listener fires, the count for "my_uwu_counter" is incremented by one.

Roles​

Defines the roles that can increment the counter.

This argument takes a role mention value.

ValueMax ValuesDescription
Role Mention15Example: @testrole (interpreted as <@&929204504572862537>)

When entering the command into Discord you do not need to type the snowflake manually. Simply type @ and a list of roles will appear, continue typing a character or two and then pick the role you want from the list. Discord will transmit the snowflake behind the scenes in a way Comfy can interpret.

Example​

when reaction emote= <:UwU:840311365813010492> -> count key= my_uwu_counter trigger= 10 roles= <@&929204504572862537>

When the reaction listener fires, the count for "my_uwu_counter" is incremented by one IF the reaction author had the role <@&929204504572862537>.

Duration (Not implemented)​

Defines a duration on the listener resource being counted, for when it will no longer increment the counter.

This argument takes a duration value.

ValueMax DurationDescription
Duration1 MonthExample: 10s (10 seconds) 1m (60 seconds) 3d (3 days) 1w (1 week) 1m (1 month)

Limitations​

Currently, the count action can only be tied to a reaction listener. In the future this limitation will be removed.
