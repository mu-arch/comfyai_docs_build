The format of when commands

Formatting "When Commands"​

When commands are simple statements that declare which actions run in response to some event happening.

If you haven't read the Introduction to automation yet, read that first.

A Complete Example​

Before we break down the parts of a When Command let's look at a real one that could be used.

Now here's how that command looks when it's actually processed by Comfy. Notice how the actual role names were converted to strange numbers. You don't need to worry about that, but we just wanted to point it out so you don't get confused.

when role has= <@&869466780156964915>, <@&869683374619508736> -> message content= "{listener.username} has triggered the listener!" channel= <#918390434848464907>

Command breakdown​

At the highest level, every command has two parts, a listener and an action.

Listeners and actions are seperated by the -> text.

when listener -> action

In the first example command role is our listener and message is our action:

when role -> message

Almost every listener and action take "arguments", arguments are special words followed by an equals sign = that indicate additional information about the listener or action.

Listener breakdown​

In the first example command our listener role has one argument has:

has= <@&869466780156964915>, <@&869683374619508736>

The has argument takes role mentions as values to tell the role listener specifically what to listen for, rather than listening for all role changes.

Action breakdown​

Our action in the example command has two arguments content and channel:

content= "{listener.username} has triggered the listener!" channel= <#918390434848464907>

content indicates the textual content that will be posted in the message, and channel indicates which channel the message will be posted to.

Pretty simple right!? Using these simple commands you can pretty much automate everything you need to on your Discord server.

Additional information​

You may have noticed the {listener.username} text inside the content argument of the message action. This is called a template. Templates can be used to suck information out of the listener's "caller". A caller is the thing that initiated some event. For example the caller of a message, role change, or reaction is the person that did that action or had that action done upon them.

The caller in this case is the person that had their roles updated. This listener makes this value available to ComfyAI by sending listener.username, which let's us replace that template with the user's actual username.
